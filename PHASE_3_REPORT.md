# PHASE 3 — Proposal Builder + activation. Отчёт

> **Отмашка (04.07.2026):** DoD Фазы 3 принят владельцем. Билд/lint/typecheck/test зелёные (61 тест). Это закрывает пункт 1 гейта входа в Фазу 4 (см. раздел «Вход Б» в `CLAUDE.md`). Незакрытого долга Фаз 0–3, блокирующего Фазу 4, не выявлено.

## Что сделано

- **Расчёт цены** `lib/pricing/calc.ts`: диапазон = base × area × (сложность × срочность × пакет), с прозрачной разбивкой (`factors` — «почему цена такая») и оценкой срока в неделях. Детерминированно, деньги — integer в рублях. Покрыт unit-тестами.
- **Режим «без цены»**: при `designers.pricing = null` (или отсутствии площади) блок стоимости в КП не собирается; на странице — пометка «режим без цены».
- **Черновик КП** `lib/proposal/build.ts` + `/dashboard/projects/[id]/proposal`:
  - Секции: задача клиента · состав работ · этапы и сроки · стоимость (если есть) · что входит · что не входит · лимиты правок · что нужно от клиента · условия завершения этапов.
  - Предзаполнение: из паспорта (задача, состав), принятых risk-карточек (`proposal_implication` → «что входит»), `proposal_defaults` (exclusions, лимит правок, условия этапов).
  - Каждая секция — **редактируемый** textarea; сохранение — server action `saveProposal`.
  - При первом входе создаётся черновик: `public_token`, статус `proposal_draft`, событие `proposal_created`.
- **Публичная страница** `/p/[public_token]` (доступ по токену через service role) + **версия для печати** через `@media print` (`.no-print` скрывает управление) — это и есть «PDF» v0.1.
- **Кнопка «Отправить»** → статус проекта `proposal_sent`, статус КП `sent`, `sent_at`, событие `proposal_sent` (activation-метрика).
- **Экран Setup** `/dashboard/setup`: профиль + pricing-мастер (базовая ставка, множители сложности/срочности/пакета, тумблер «считать цену») + шаблон КП (exclusions, лимит правок, условие этапа). Server action `saveSetup`.

## События (метрики валидации)

Все пишутся в `events`: `intake_link_created`, `brief_started`, `brief_completed`, `proposal_created`, `proposal_sent`. Метрика activation (№2 из стратегии) — наблюдаемое событие `proposal_sent`.

## Тесты

- `calcPrice` — детерминированные диапазоны, множители, срок.
- `buildProposalSections` — наличие/отсутствие блока цены (режим «без цены»), вплетение принятых карточек в «что входит», предзаполнение задачи из паспорта, полный набор секций.

## Как проверить руками за 5 минут

Сквозной сценарий — в `DEMO.md`. Кратко:
1. Setup → задать ставки (или выключить «считать цену»).
2. Пройти бриф → принять карточки на Review Board.
3. «Собрать коммерческое предложение» → отредактировать секции → «Сохранить».
4. Открыть публичную ссылку `/p/[token]` → «Печать / PDF».
5. «Отправить» → статус `proposal_sent`, событие в `events`.

## Известные ограничения

- Сложность стиля для расчёта по умолчанию `mid`, пакет — `full` (в quick brief уровень сервиса не спрашивается). Дизайнер правит текст блока цены вручную; интерактивный пересчёт по сложности/пакету прямо в билдере — кандидат в v0.2 (BACKLOG).
- Одна версия КП (`version=1`); версионирование правок — позже.
- «PDF» — печатная HTML-страница (guardrail: без PDF-библиотек).
