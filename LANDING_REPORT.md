# LANDING_REPORT — Кинематографичный сайт ARHIDOM (05.07.2026)

## Что сделано

Полная замена публичной части сайта на кинематографичный тёмный лендинг
(уголь · слоновая кость · бронза · олива, Cormorant Garamond + Golos Text) —
без изменения приложения: auth, кабинет, брифы, КП и данные не тронуты.

### Продуктовые изменения (не только маркетинг)

- **Ответ клиента на публичном КП** (`/p/[token]`): CTA «Принять / Обсудить /
  Запросить правки». Без миграций: ответ = событие в `events`
  (`proposal_accepted` / `proposal_discussion_requested` /
  `proposal_changes_requested`), первый ответ финальный. + событие
  `proposal_viewed` (один раз). Дизайнер видит бейджи «Клиент открывал КП» и
  «Ответ клиента: …» на странице КП в кабинете. Это закрывает пункты S3/S4
  «Лестницы запуска» в минимальном виде.
- **PWA-промпт больше не показывается** на `/i/*`, `/b/*`, `/p/*`, `/demo*`.
- **Ребрендинг**: `ru.app.name` = «ARHIDOM» (единая точка — i18n).
- **`/api/pilot`**: событие `pilot_request` (rate limit 10/IP/час). Содержимое
  заявки уходит письмом (mailto на `NEXT_PUBLIC_SUPPORT_EMAIL`) или копируется —
  таблицы лидов в v0.1 нет, PII не сохраняем.

### Сгенерированные медиа (Higgsfield)

Видео — Cinema Studio Video 3.0; изображения — Nano Banana. Все — тёмная
архитектурная сцена, данные на визуалах нечитаемые/замазанные. CDN по
умолчанию; `npm run fetch:media` + `NEXT_PUBLIC_MEDIA_BASE=/landing` — свой
хостинг. Карта: `components/landing/media.ts`.

- hero-видео 10 c 16:9 + постер-фолбэк (hero)
- туман → структура (секция 2) · паспорт 4:5 (sticky-сцена) · риск-карточки
  вокруг плана · сборка КП · Review Board · клиентский бриф · pricing ·
  «одна ссылка — одна студия» · данные · повестка встречи

### Техника

- Без новых зависимостей: reveal — IntersectionObserver, sticky-сцены и
  параллакс — rAF (`components/landing/cinema.tsx`); `prefers-reduced-motion`
  уважается (видео скрывается, остаётся постер; анимации отключаются).
- На мобильных sticky-сцены заменяются статичными версиями (не клипуются).
- Все строки — в `lib/i18n/ru.ts` (`ru.landing`).
- `npm run build / lint / typecheck / test` — зелёные (55 тестов).

## Как проверить руками за 5 минут

1. `npm run dev` → `/` — hero-видео, скролл до низа: sticky-сцены, секции.
2. «Попробовать бриф глазами клиента» → `/demo/brief` → пройти → финал →
   «Создать настоящий бриф» создаёт реальный клиентский бриф (existing flow).
3. `/demo/proposal` → нажать «Принять предложение» (демо, ничего не шлёт).
4. Настоящий цикл: кабинет → проект → КП → «Отправить» → открыть `/p/<token>`
   → ответить → вернуться в кабинет: бейдж «Ответ клиента».
5. `/pilot` → заполнить форму → «Отправить» → письмо/копия текста заявки.
6. Мобильная ширина: меню-бургер, статичные версии sticky-секций.

## Ограничения и блокеры (для владельца)

- **Юрблок**: `/legal/*` — честные описания практик пилота, НЕ полная
  юридическая редакция (нет реквизитов оператора — их не выдумываем).
  Остаётся блокером публичного запуска (стадия L1).
- **Чекбокс согласия в брифе** (P3) — не входил в этот диф; на брифе есть
  privacy-заметка, явный чекбокс — следующий шаг.
- **Ответ клиента по КП** хранится событиями, `proposals.status` не расширялся
  (без миграций). Полноценные статусы accepted/declined — отдельная миграция.
- **Заявка на пилот** не сохраняет содержимое в БД (только факт события) —
  доставка через письмо. Таблица лидов — кандидат в BACKLOG.
- Медиа живут на CDN Higgsfield — для независимости прогнать
  `npm run fetch:media` и выставить `NEXT_PUBLIC_MEDIA_BASE=/landing` на Vercel.
