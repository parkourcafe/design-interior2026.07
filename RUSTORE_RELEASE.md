# RUSTORE_RELEASE — готовность ARHIDOM к запуску в RuStore

Обновлено: 09.07.2026. Цель: подготовить карточку, технические требования и
owner-действия для первой отправки ARHIDOM в RuStore как Android TWA.

## 1. Короткий статус

**Техническая готовность к сборке APK/AAB: высокая.** Сайт уже PWA/TWA-ready:
manifest, service worker, иконки, Digital Asset Links endpoint, HTTPS, privacy
и terms страницы есть.

**Готовность к отправке в RuStore: условная, примерно 75%.** Кодовая часть
готова, но владелец должен закрыть внешние блокеры: аккаунт разработчика,
package name, keystore, APK/AAB, отпечаток SHA-256 в Vercel, тестовый аккаунт
для модератора, реальные скриншоты, рабочий email поддержки и реквизиты
оператора ПДн в юридическом блоке.


Итог: build/lint/typecheck/test зелёные, 55 unit-тестов прошли,
production audit — 0 vulnerabilities.

Store-critical endpoints проверены локально через `next start`:

- `/manifest.webmanifest` → `200 application/manifest+json`;

production bundle/TWA, но стоит обновить отдельной задачей.

## 5. Owner-блокеры перед отправкой

1. Создать/подтвердить аккаунт разработчика RuStore.
2. Выбрать package name, например `space.arhidom.twa`. После публикации не менять.
3. Собрать APK/AAB через PWABuilder или Bubblewrap.
4. Сохранить keystore, alias и пароли в password manager/offline backup.
5. Получить SHA-256 отпечаток сертификата RuStore.
6. В Vercel Production env задать:

```bash
ANDROID_PACKAGE_NAME=space.arhidom.twa
ANDROID_CERT_SHA256=<SHA-256 отпечаток RuStore>
```

Если позже будет Google Play, добавить второй SHA-256 через запятую.

7. Проверить:

```bash
https://arhidom.space/.well-known/assetlinks.json
https://arhidom.space/manifest.webmanifest
https://arhidom.space/icons/512
```

8. Подготовить тестовый аккаунт для модератора или описать путь проверки через `/demo`.
9. Снять реальные скриншоты установленной TWA.
10. Проверить, что `NEXT_PUBLIC_SUPPORT_EMAIL` реально принимает письма.
11. Добавить реквизиты оператора ПДн в `/legal/privacy` и `/legal/terms` до массового публичного запуска.

## 6. Декларация данных для RuStore

Заполнять честно по фактическому поведению продукта.

Категории данных:

- Контактные данные: email, телефон, соцсети/мессенджер клиента или дизайнера.
- Пользовательский контент: ответы брифа, референсы, комментарии, загруженные планы/фото.
- Данные аккаунта: email дизайнера, профиль студии, настройки цены.
- Диагностические/событийные данные: события воронки (`brief_started`, `proposal_sent` и т.п.).

Цели:

- предоставление сервиса;
- авторизация дизайнера;
- формирование паспорта проекта, risk cards, цены и КП;
- поддержка и безопасность.

Передача третьим лицам:

- Supabase как инфраструктура хранения/авторизации;
- LLM-провайдер из env для AI-анализа, при этом город и контакты маскируются до вызова модели.

Удаление:

- по запросу через поддержку; на этапе пилота вручную.

Разрешения Android:

- ожидаемо без чувствительных разрешений. После загрузки APK проверить список в консоли RuStore. Если консоль обнаружит чувствительные разрешения, каждое нужно обосновать отдельно.

## 7. Карточка RuStore — готовые поля

**Название:** ARHIDOM

**Категория:** Бизнес-сервисы  
Если такой категории нет в консоли: Полезные инструменты.

**Возраст:** 0+

**Краткое описание (до 80 символов):**

AI-пресейл для дизайнеров: бриф, паспорт проекта, риски, цена и КП.

**Полное описание:**

ARHIDOM — рабочий инструмент интерьерного дизайнера и студии для этапа до начала проекта.

Дизайнер отправляет клиенту ссылку на бриф. Клиент отвечает без регистрации, в спокойном брендированном интерфейсе. ARHIDOM превращает ответы в паспорт проекта, находит возможные риски, помогает рассчитать стоимость по правилам студии и собирает черновик коммерческого предложения.

Что доступно:

- клиентский бриф без регистрации;
- паспорт проекта из ответов клиента;
- карточки рисков по бюджету, срокам, стилю, функции и техническим ограничениям;
- расчёт вилки цены по логике студии;
- конструктор КП и публичная ссылка на предложение;
- ответ клиента на КП: принять, обсудить или запросить правки.

AI предлагает — дизайнер утверждает. Решение всегда остаётся за дизайнером.

Идёт закрытый пилот, пользование бесплатное.

**Что нового в версии 1.0.0:**

Первый релиз ARHIDOM в RuStore: клиентский бриф, паспорт проекта, карточки рисков, расчёт цены и коммерческое предложение в одном пресейл-контуре.

## 8. Комментарий модератору RuStore

ARHIDOM опубликован как Trusted Web Activity: Android-приложение открывает полноэкранную PWA-версию сервиса на домене arhidom.space. Это не редирект на рекламный сайт, а рабочий продукт: внутри доступны публичная главная, демо-бриф, демо-КП, юридические страницы и кабинет дизайнера.

Для проверки без регистрации:

1. Откройте приложение.
2. Перейдите в демо-бриф через кнопку на главной или URL `/demo/brief`.
3. Пройдите демо-бриф.
4. Откройте демо-КП `/demo/proposal`.

Для проверки кабинета используйте тестовый аккаунт:

- Email: `<вставить email>`
- Пароль: `<вставить пароль>`

Если тестовый аккаунт не нужен, удалите последние две строки перед отправкой.

## 9. Скриншоты для RuStore

Подготовить 5–6 PNG/JPG, 9:16, например 1080×1920, каждый ≤3 МБ:

1. Главная ARHIDOM.
2. Демо-бриф: вопрос клиента.
3. Паспорт проекта / структура ответов.
4. Risk cards / Review Board.
5. Коммерческое предложение.
6. Ответ клиента на КП.

Скриншоты должны показывать реальный интерфейс, а не только промо-баннеры.
