# PHASE 2 — Review Board. Отчёт

## Что сделано

Страница проекта у дизайнера `/dashboard/projects/[id]`:

- **Паспорт в человеко-читаемом виде** — компонент `components/passport-view.tsx`: объект, горизонт актива, домохозяйство, образ жизни, бюджет, сроки, стиль (референсы/анти/заметки), pain points. Все подписи из `lib/i18n/ru.ts`.
- **Карточки рисков — все 6 полей** (`app/dashboard/projects/[id]/review.tsx`): тип риска, evidence, impact, confidence, действие дизайнера, следствие для КП; тег источника (правило/AI). Тон строго из CLAUDE.md («Возможный риск…»). Кнопки **принять / отклонить** (server action `setCardStatus`, статус пишется в `risk_cards.status`).
- **Блок «недостающие данные»** — `missingFields(passport)`: пустые ключевые поля паспорта.
- **Блок «вопросы к первой встрече»** — `firstMeetingQuestions(cards)`: собирается из `designer_action` **принятых** карточек (обновляется по мере принятия).
- **Кнопка «Пересобрать карточки (AI)»** — server action `rerunRisks`: перечитывает ответы, гоняет пайплайн заново, заменяет карточки.
- **Обработка деградации:** если все карточки `source='rule'` — показывается баннер «AI-анализ временно недоступен — показаны карточки на основе правил», интерфейс работает полностью. Пайплайн никогда не роняет страницу (rule-карточки — всегда доступный фолбэк).

## Тесты

- `missingFields` — пустые/заполненные поля.
- `firstMeetingQuestions` — берёт `designer_action` только из принятых карточек.

## Как проверить руками за 5 минут

1. С реальными env пройти бриф по intake-ссылке (Фаза 1).
2. Открыть `/dashboard/projects/[id]` — увидеть паспорт, карточки, недостающие данные.
3. Принять 1–2 карточки → в блоке «вопросы к первой встрече» появятся их `designer_action`.
4. Нажать «Пересобрать карточки (AI)» — карточки перегенерируются.
5. Если LLM не сконфигурирован — вверху виден баннер деградации, но всё работает.

## Известные ограничения

- Баннер деградации показывается по эвристике «все карточки — rule» (флаг `llmOk` не персистится отдельной колонкой в v0.1). При легитимном отсутствии LLM-рисков баннер тоже покажется — безопасная сторона ошибки.
- Отклонённые карточки просто помечаются `rejected` (обратная связь для будущей калибровки онтологии — по стратегии; сбор этой обратной связи отдельной аналитикой — в BACKLOG).
